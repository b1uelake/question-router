# Plan Contract

Use for roadmaps, learning plans, project plans, workflows, execution sequences, and "from zero to one" requests.

## Classify As Plan When

- The user asks "how do I learn", "build a plan", "roadmap", "from 0 to 1", "step by step", or wants a sequence to an outcome.
- The main value is prioritization, sequencing, dependencies, and checkpoints.
- The answer should turn a goal into action.

## Ask First When

Ask 1-5 grouped questions when the plan is personalized, long-horizon, or broad:

- Current baseline across the relevant stack, not just one skill.
- Weekly time, pace, and deadline.
- Target outcome: project, job readiness, internal work, conceptual mastery, or portfolio.
- Constraints: budget, paid APIs, hardware, preferred stack, language, operating system, and tools.
- Learning preference and proof of progress: reading, coding, projects, exercises, notes, tests, portfolio, or review checkpoints.

For 4+ week learning plans, career transitions, and broad engineering roadmaps, ask intake before writing the plan unless already provided.

If the user declines intake, state default assumptions and label the plan as a default track, not a personalized plan.

## Answer Contract

Include:

- Target outcome.
- Assumptions and constraints.
- Phased steps in priority order.
- Dependencies and sequencing logic.
- Risks and mitigations.
- Checkpoints or success criteria.
- Immediate next step.

## Quality Rules

- Avoid vague phases like "research, execute, optimize" without concrete work.
- Avoid overplanning when the user needs a first move.
- Tie every phase to a deliverable or proof of progress.
- For learning plans, include practice loops and checkpoints, not only topics.

## Example Routes

- "Build me a 10-week study plan for AI application engineering" -> plan + learning; ask intake across Python, web/API, LLM, RAG, deployment, time, goals, constraints, and proof.
- "How do I build a high-quality website from zero to one?" -> plan + technical; clarify purpose, audience, content, stack, quality bar, and launch constraints.
- "Give me a 30-day writing plan" -> plan; ask target audience/output and available time if missing.
